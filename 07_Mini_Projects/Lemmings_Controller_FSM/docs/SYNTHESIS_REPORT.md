# Synthesis Report

## Design Information

- **Project:** Lemmings Controller FSM
- **RTL Language:** Verilog HDL
- **Top Module:** `lemmings`

---

## Purpose

The synthesis step was performed to:

- Verify that the RTL is synthesizable.
- Inspect the inferred hardware resources.
- Generate a gate-level netlist.
- Visualize the synthesized hardware implementation.

## Synthesis Flow

The RTL was synthesized using the following Yosys flow:

```yosys
read_verilog lemmings.v
hierarchy -check -top lemmings
proc
opt
fsm
stat
write_verilog synthesized.v
show -format svg -prefix lemmings
```

## Synthesis Observations

- RTL synthesized successfully and produced a valid gate-level netlist.
- No unintended latches were inferred.
- Two asynchronous reset D flip-flops were inferred:
  - State register
  - Fall counter
- The design implements an **FSM with datapath** because the next-state logic depends on a fall-duration counter.
- Consequently, Yosys did not automatically extract the FSM during the `fsm` pass, reporting:
  > *Circuit seems to be self-resetting.*

This behavior is expected for this implementation.
